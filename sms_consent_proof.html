SMS Consent Documentation — Personal AI Phone Assistant

Service Description

This is a personal AI phone assistant that makes outbound calls on behalf of its user (e.g., scheduling appointments, making reservations). SMS is used as the input/output interface: the user texts a request, and the service texts back a confirmation and eventual outcome summary.

Consent Model: User-Initiated

All messaging is initiated by the user. The service never sends unsolicited messages. The only SMS messages sent by the service are direct responses to user-initiated requests:

Message TypeDirectionTrigger
Call requestUser → ServiceUser sends SMS with instructions
AcknowledgmentService → UserImmediate reply to user's request
Call outcome summaryService → UserAfter the requested call completes

Example Interaction Flow

User initiates:
Call 555-123-4567 and schedule a dental cleaning for next Tuesday
Service acknowledges:
Got it! Calling +15551234567 now. I'll text you when it's done.
Service reports outcome:
Call complete: Dental cleaning scheduled for Tues May 5 at 10am with Dr. Smith

Key Points

  • Messages are only sent to the same phone number that initiated the request
  • No marketing, promotional, or broadcast messaging
  • No third-party recipients — all messages go back to the requesting user
  • Only authorized phone numbers (pre-configured by the service owner) can trigger calls
  • The service is for personal use only
